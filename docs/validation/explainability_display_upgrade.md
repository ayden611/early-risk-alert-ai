# Explainability Display Upgrade — Validation Intelligence

## Purpose

This upgrade addresses score saturation concerns by changing the display emphasis from raw peak risk score alone to clinical prioritization context.

## What Changed

The Validation Intelligence page now displays:

- Priority tier distribution
- Top contributing drivers
- Representative detected review examples
- Queue rank
- Lead time before event
- Primary driver
- Score
- Hospital-safe display language

## Why This Matters

If many patients reach a high score, a hospital reviewer may ask how the system distinguishes urgent from most urgent. The answer should not rely on raw score alone.

ERA should foreground:

- Priority Tier
- Queue Rank
- Primary Driver
- Trend Context
- First Threshold Crossing
- Lead Time Before Event

## Pilot-Safe Language

Use:

- “May warrant review”
- “Primary driver”
- “Priority tier”
- “Retrospective timing analysis”
- “Decision support only”

Avoid:

- “Diagnosis”
- “Cause”
- “Requires intervention”
- “Proven early detection”
- “Autonomous escalation”

## Decision-Support Notice

Decision support only. Not intended to diagnose, direct treatment, replace clinician judgment, or independently trigger escalation.
