Validate ERA CSV or Convert Raw MIMIC / De-Identified Monitoring Exports

This page gives you one controlled ingest workflow for retrospective validation. Use the left side for ERA-format CSV files. Use the right side for raw MIMIC-style files or raw de-identified exports that need conversion before analysis.

De-identified data only. Do not upload names, MRNs, dates of birth, or direct identifiers. ERA CSV validation and raw-data conversion are both retrospective workflows for controlled evaluation and independent review.
Mode A

ERA CSV → Analyze

Upload a de-identified ERA-format CSV or CSV.GZ file and run retrospective validation directly.

Download Schema Template
Use this for hospital de-identified data that already matches the ERA schema: patient_id, timestamp, heart_rate, spo2, bp_systolic, bp_diastolic, respiratory_rate, temperature_f.
Mode B

Raw MIMIC / Raw Export → Convert + Analyze

Upload raw MIMIC-style monitoring files or similarly structured de-identified exports. The platform converts them to ERA format, then runs retrospective analysis in one step.

Required inputs: chartevents + icustays. Optional: datetimeevents for event-window flagging. This is the correct path for raw MIMIC data and raw de-identified exports that do not already match the ERA schema.
